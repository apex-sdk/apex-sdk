# API Reference: System Chain Standard

**Apex SDK is the canonical System Chain Standard Library for Polkadot Asset Hub, Revive, and PolkaVM/Solidity.**

*Difficulty: Intermediate â€¢ Estimated read time: 15-20 minutes*


## Core Classes

The Apex SDK provides unified modules for system chain operations:

### AssetManager

High-level API for Polkadot Asset Hub:

```rust
use apex_sdk::substrate::AssetManager;
let asset_mgr = AssetManager::connect_env().await?;
let assets = asset_mgr.list_assets().await?;
```

### ReviveAdapter

Native Revive protocol integration:

```rust
use apex_sdk::revive::ReviveAdapter;
let revive = ReviveAdapter::connect_env().await?;
let recoverable = revive.list_recoverable_assets().await?;
```

### ContractManager

PolkaVM/Solidity contract deployment and calls:

```rust
use apex_sdk::polkavm::{ContractManager, ContractConfig};
let contract = ContractManager::deploy(&bytecode, ContractConfig::default(), &deployer_account).await?;
let result = contract.call_method("transfer", &[recipient, amount]).await?;
```

**Syntax:**
```rust
pub async fn new(config: NetworkConfig) -> Result<Self, ApexError>
```

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| config | NetworkConfig | âœ… | Network configuration specifying chain, RPC URL, and other connection parameters |

**Return Value:** Returns `Result<ApexClient, ApexError>` - A new client instance or an error if initialization fails.

**Exceptions:**
- `ApexError::NetworkError` - Invalid RPC URL or network unreachable
- `ApexError::ConfigError` - Invalid configuration parameters
- `ApexError::AuthError` - API key authentication failed

## Network Operations

### get_chain_id()

Retrieves the chain ID of the connected network.

```rust
let chain_id = client.get_chain_id().await?;
println!("Connected to chain: {}", chain_id);

// Chain ID constants
match chain_id {
    1 => println!("Ethereum Mainnet"),
    137 => println!("Polygon Mainnet"),
    56 => println!("Binance Smart Chain"),
    _ => println!("Unknown chain: {}", chain_id),
}
```

**Syntax:**
```rust
pub async fn get_chain_id(&self) -> Result<u64, ApexError>
```

**Return Value:** Returns `Result<u64, ApexError>` - The chain ID as an unsigned 64-bit integer.

**Exceptions:**
- `ApexError::NetworkError` - Network connection failed
- `ApexError::RpcError` - RPC call failed

**Syntax:**

**Return Value:** Returns `Result<u64, ApexError>` - The latest block number.

**Exceptions:**

- `ApexError::RpcError` - RPC call failed

### get_balance()

Retrieves the balance of a specific address.


## Transaction Operations

### send_transaction()

Sends a transaction to the blockchain.

```rust
use apex_sdk::types::{Address, U256, TransactionRequest};
use apex_sdk::wallet::Wallet;

// Create wallet from private key
let wallet = Wallet::from_private_key("0x...")?;

// Build transaction
let to_address = Address::from_str("0x742d35Cc6634C0532925a3b8D93EB2a1732B6123")?;
let amount = U256::from_dec_str("1000000000000000000")?; // 1 ETH

let tx_request = TransactionRequest::new()
    .to(to_address)
    .value(amount)
    .gas_limit(21000)
    .gas_price(U256::from(20_000_000_000u64)); // 20 gwei

// Sign and send
let tx_hash = client
    .send_transaction(&tx_request, &wallet)
    .await?;

println!("Transaction sent: {}", tx_hash);

// Wait for confirmation
let receipt = client.wait_for_transaction(&tx_hash).await?;
println!("Transaction confirmed in block: {}", receipt.block_number);
```

**Syntax:**
```rust
pub async fn send_transaction(
    &self, 
    request: &TransactionRequest, 
    wallet: &Wallet
) -> Result<H256, ApexError>
```

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| request | &TransactionRequest | âœ… | Transaction details including recipient, amount, and gas parameters |
| wallet | &Wallet | âœ… | Wallet instance for signing the transaction |

**Return Value:** Returns `Result<H256, ApexError>` - The transaction hash.

**Exceptions:**
- `ApexError::InsufficientFunds` - Account doesn't have enough balance
- `ApexError::GasEstimationFailed` - Could not estimate gas limit
- `ApexError::TransactionFailed` - Transaction execution failed
- `ApexError::NetworkError` - Network connection failed

## Smart Contract Interaction

### Contract Deployment

Deploy smart contracts to the blockchain.

```rust
use apex_sdk::contract::{Contract, ContractDeployment};
use apex_sdk::abi::Abi;

// Load contract ABI and bytecode
let abi = Abi::from_json(include_str!("./MyContract.abi.json"))?;
let bytecode = include_str!("./MyContract.bin");

// Create deployment
let deployment = ContractDeployment::new(abi, bytecode)
    .with_constructor_args(&[
        Token::Uint(U256::from(1000000)), // Initial supply
        Token::String("MyToken".to_string()), // Token name
    ])
    .with_gas_limit(2_000_000)
    .build()?;

// Deploy contract
let contract_address = client
    .deploy_contract(&deployment, &wallet)
    .await?;

println!("Contract deployed at: {}", contract_address);

// Create contract instance for interaction
let contract = Contract::new(contract_address, abi, &client);
```

### Contract Method Calls

Interact with deployed smart contracts.

```rust
// Read-only method call (view function)
let balance: U256 = contract
    .method("balanceOf")?
    .call(&[Token::Address(wallet.address())])
    .await?
    .into();

println!("Token balance: {}", balance);

// State-changing method call
let tx_hash = contract
    .method("transfer")?
    .send(&[
        Token::Address(to_address),
        Token::Uint(U256::from(1000)),
    ], &wallet)
    .await?;

println!("Transfer transaction: {}", tx_hash);

// Call with custom gas settings
let tx_hash = contract
    .method("complexOperation")?
    .gas_limit(500_000)
    .gas_price(U256::from(25_000_000_000u64))
    .send(&[Token::String("parameter".to_string())], &wallet)
    .await?;
```

**Method Parameters:**

| Parameter | Type | Required | Description | Default |
|-----------|------|----------|-------------|---------|
| method_name | &str | âœ… | Name of the contract method to call | - |
| args | &[Token] | âŒ | Array of method arguments encoded as ABI tokens | Empty array |
| wallet | &Wallet | âœ… | Wallet for signing state-changing transactions (send only) | - |

## Cross-Chain Operations

### Cross-Chain Bridge

Transfer assets between different blockchain networks.

```rust
use apex_sdk::bridge::{BridgeConfig, CrossChainTransfer};

// Configure bridge settings
let bridge_config = BridgeConfig::new()
    .with_ethereum_bridge("0x...") // Ethereum bridge contract
    .with_polygon_bridge("0x...") // Polygon bridge contract
    .with_confirmation_blocks(12) // Wait for confirmations
    .build()?;

// Create cross-chain client
let cross_client = client.with_bridge(bridge_config).await?;

// Initiate cross-chain transfer
let transfer = CrossChainTransfer::new()
    .from_chain(Chain::Ethereum)
    .to_chain(Chain::Polygon)
    .token_address("0x...") // ERC-20 token address
    .amount(U256::from_dec_str("1000000000000000000")?) // 1 token
    .recipient("0x...") // Recipient on destination chain
    .build()?;

// Execute transfer
let transfer_id = cross_client
    .execute_transfer(&transfer, &wallet)
    .await?;

println!("Cross-chain transfer initiated: {}", transfer_id);

// Monitor transfer status
let status = cross_client
    .get_transfer_status(&transfer_id)
    .await?;

match status {
    TransferStatus::Pending => println!("Transfer pending"),
    TransferStatus::Confirmed => println!("Transfer confirmed"),
    TransferStatus::Completed => println!("Transfer completed"),
    TransferStatus::Failed(error) => println!("Transfer failed: {}", error),
}
```

**Cross-Chain Parameters:**

| Parameter | Type | Required | Description | Default |
|-----------|------|----------|-------------|---------|
| from_chain | Chain | âœ… | Source blockchain network | - |
| to_chain | Chain | âœ… | Destination blockchain network | - |
| token_address | Address | âœ… | Token contract address on the source chain | - |
| amount | U256 | âœ… | Amount to transfer in smallest token units | - |
| recipient | Address | âœ… | Recipient address on the destination chain | - |
| gas_limit | u64 | âŒ | Custom gas limit for the transfer transaction | Auto-estimated |

## Event Monitoring

### Event Listening

Monitor blockchain events in real-time.

```rust
use apex_sdk::events::{EventFilter, EventStream};
use futures::StreamExt;

// Create event filter for ERC-20 transfers
let transfer_filter = EventFilter::new()
    .address("0x...") // Token contract address
    .event("Transfer(address,address,uint256)")
    .from_block(18000000)
    .build()?;

// Create event stream
let mut event_stream = client
    .subscribe_events(&transfer_filter)
    .await?;

// Process events
while let Some(event) = event_stream.next().await {
    match event {
        Ok(log) => {
            println!("Transfer event:");
            println!("  From: {}", log.topics[1]);
            println!("  To: {}", log.topics[2]);
            println!("  Amount: {}", log.data);
            println!("  Block: {}", log.block_number);
        }
        Err(error) => {
            eprintln!("Event error: {}", error);
        }
    }
}
```

### Historical Event Queries

Query past events from the blockchain.

```rust
// Query events from a specific block range
let events = client
    .get_events(&transfer_filter)
    .from_block(18000000)
    .to_block(18001000)
    .limit(100)
    .execute()
    .await?;

println!("Found {} transfer events", events.len());

for event in events {
    let decoded = contract.decode_event("Transfer", &event)?;
    println!(
        "Transfer: {} -> {} ({})",
        decoded.from,
        decoded.to,
        decoded.value
    );
}
```

## Error Handling

### ApexError Types

The SDK uses a comprehensive error system for robust error handling.

```rust
use apex_sdk::error::{ApexError, ErrorKind};

async fn handle_transaction() -> Result<(), ApexError> {
    match client.send_transaction(&tx_request, &wallet).await {
        Ok(tx_hash) => {
            println!("Transaction successful: {}", tx_hash);
            Ok(())
        }
        Err(ApexError::InsufficientFunds { required, available }) => {
            eprintln!("Insufficient funds: need {}, have {}", required, available);
            Err(ApexError::InsufficientFunds { required, available })
        }
        Err(ApexError::NetworkError(msg)) => {
            eprintln!("Network error: {}", msg);
            // Retry logic could go here
            Err(ApexError::NetworkError(msg))
        }
        Err(ApexError::TransactionFailed { tx_hash, reason }) => {
            eprintln!("Transaction {} failed: {}", tx_hash, reason);
            Err(ApexError::TransactionFailed { tx_hash, reason })
        }
        Err(error) => {
            eprintln!("Unexpected error: {}", error);
            Err(error)
        }
    }
}
```

**Common Error Types:**

| Error Type | Description |
|------------|-------------|
| NetworkError | Network connectivity or RPC endpoint issues |
| InsufficientFunds | Account balance insufficient for transaction |
| TransactionFailed | Transaction execution failed on-chain |
| InvalidAddress | Malformed or invalid Ethereum address |
| ContractError | Smart contract call or deployment error |
| ConfigError | Invalid configuration parameters |

## Performance & Best Practices

### Connection Pooling

Optimize performance with connection pooling and batching.

```rust
use apex_sdk::pool::{ConnectionPool, PoolConfig};

// Create connection pool
let pool_config = PoolConfig::new()
    .max_connections(10)
    .min_connections(2)
    .connection_timeout(Duration::from_secs(30))
    .idle_timeout(Duration::from_secs(600))
    .build()?;

let pool = ConnectionPool::new(network_config, pool_config).await?;

// Batch multiple requests
let batch = client.batch()
    .add_call("eth_getBalance", &[address1])
    .add_call("eth_getBalance", &[address2])
    .add_call("eth_getBlockNumber", &[])
    .execute()
    .await?;

let balance1: U256 = batch.result(0)?;
let balance2: U256 = batch.result(1)?;
let block_number: u64 = batch.result(2)?;
```

### Gas Optimization

Manage gas costs effectively.

```rust
use apex_sdk::gas::{GasEstimator, GasPrice};

// Estimate gas for transaction
let gas_estimate = client
    .estimate_gas(&tx_request)
    .await?;

// Get current gas prices
let gas_price = client.get_gas_price().await?;
let gas_price_gwei = gas_price.as_gwei();

println!("Estimated gas: {}", gas_estimate);
println!("Gas price: {} gwei", gas_price_gwei);

// Use EIP-1559 for better gas management
let tx_request = TransactionRequest::new()
    .to(address)
    .value(amount)
    .max_fee_per_gas(gas_price * 120 / 100) // 20% above base fee
    .max_priority_fee_per_gas(U256::from(2_000_000_000u64)) // 2 gwei tip
    .gas_limit(gas_estimate * 110 / 100) // 10% buffer
    .build()?;
```
