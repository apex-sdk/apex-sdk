### 2. Environment Setup

Create `.env` file for sensitive configuration:

```bash
# Network RPC URLs (never commit to git!)
ETHEREUM_RPC_URL=https://eth-mainnet.g.alchemy.com/v2/YOUR-API-KEY
POLYGON_RPC_URL=https://polygon-rpc.com

# Wallet Configuration
PRIVATE_KEY=your-private-key-here
MNEMONIC="your twelve word mnemonic phrase here"

# Optional: Additional Services
INFURA_PROJECT_ID=your-infura-project-id
ALCHEMY_API_KEY=your-alchemy-api-key
```

> **Security Warning:** Never commit your `.env` file to version control! Add it to your `.gitignore`:
> 
> ```bash
> echo ".env" >> .gitignore
> ```

**Environment Variables Reference:**

| Variable | Required | Description |
|----------|----------|-------------|
| `ETHEREUM_RPC_URL` | Yes | Ethereum network RPC endpoint |
| `POLYGON_RPC_URL` | No | Polygon network RPC endpoint |
| `PRIVATE_KEY` | Yes* | Wallet private key (hex format) |
| `MNEMONIC` | Yes* | 12/24 word recovery phrase |

*Either `PRIVATE_KEY` or `MNEMONIC` is required, not both.

## Your First System Chain App

Create `src/main.rs`:

```rust
use apex_sdk::prelude::*;
use apex_sdk::substrate::AssetManager;
use apex_sdk::revive::ReviveAdapter;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // Initialize Asset Hub manager
    let asset_mgr = AssetManager::connect_env().await?;
    println!("Connected to Asset Hub");

    // List all assets
    let assets = asset_mgr.list_assets().await?;
    println!("Assets: {assets:?}");

    // Use Revive protocol
    let revive = ReviveAdapter::connect_env().await?;
    let recoveries = revive.list_recoverable_assets().await?;
    println!("Recoverable: {recoveries:?}");

    Ok(())
}
```

### Parameters

- **`config_file`** - Path to TOML configuration file
- **`enable_feature`** - Feature flags to activate SDK functionality
- **`from_chain`** - Source blockchain for the transfer
- **`to_chain`** - Destination blockchain for the transfer
- **`amount`** - Transfer amount (use `Amount::from_eth()` for convenience)
- **`recipient`** - Destination wallet address

### Return Value

Returns `Result<(), Box<dyn Error>>` where:
- **`Ok(())`** - Successful execution
- **`Err(error)`** - Various error types (see Exceptions below)

### Exceptions

- **`ConfigError`** - Invalid configuration file or missing required fields
- **`NetworkError`** - RPC connection issues or network timeouts
- **`WalletError`** - Invalid private key, insufficient funds, or wallet access issues
- **`TransferError`** - Cross-chain transfer failures or validation errors

```rust
use apex_sdk::testing::*;

#[tokio::test]
async fn test_wallet_creation() {
    let apex = ApexSDK::new_test().await.unwrap();
    let wallet = apex.create_test_wallet().await.unwrap();
    
    assert!(!wallet.address().is_zero());
    assert!(wallet.private_key().len() > 0);

#[tokio::test]
async fn test_cross_chain_transfer() {
    let apex = ApexSDK::new_test().await.unwrap();
    let wallet = apex.create_funded_test_wallet().await.unwrap();

        .get_native_balance(&wallet.address(), ChainType::Ethereum)
        .await
        .unwrap();
    
    assert!(initial_balance > Amount::from_eth(1.0));
}
```

## Performance Tips

### Best Practices

- **Reuse SDK instances** - Create once, use throughout your application
- **Use connection pooling** - Enable in config for better performance
- **Batch operations** - Group multiple transactions when possible
- **Cache contract instances** - Avoid recreating for the same address
- **Monitor gas prices** - Use `apex.suggest_gas_price()` for optimal fees

### Common Mistakes to Avoid

- Don't create new SDK instances for each operation
- Don't ignore error handling - always use proper `Result<T, E>` types
- Don't hardcode addresses - use configuration files
- Don't poll unnecessarily - use event subscriptions instead
- Don't ignore network congestion indicators

## Troubleshooting

### Common Exceptions

#### Network Connection Failed

**Error:** `NetworkError: Connection timeout`

**Cause:** Invalid RPC URL or network connectivity issues

**Solution:**
```rust
// Increase timeout and add retry logic
let apex = ApexSDK::builder()
    .timeout(Duration::from_secs(60)) // Increase timeout
    .retry_config(RetryConfig {
        max_retries: 5,
        backoff_factor: 2.0,
    })
    .build()
    .await?;
```

#### Insufficient Gas

**Error:** `TransactionError: Gas estimation failed`

**Cause:** Insufficient funds or incorrect gas estimation

**Solution:**
```rust
// Use dynamic gas estimation with buffer
let gas_estimate = apex
    .estimate_gas(&transaction)
    .await?
    .checked_mul(120) // Add 20% buffer
    .ok_or("Gas estimation overflow")?;

transaction.gas_limit = gas_estimate;
```

#### Transaction Nonce Issues

**Error:** `TransactionError: Nonce too low/high`

**Cause:** Incorrect transaction nonce management

**Solution:**
```rust
// Get current nonce automatically
let nonce = apex.get_transaction_count(&wallet.address()).await?;
transaction.nonce = nonce;
```
